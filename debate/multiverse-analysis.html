Key Innovations

Stable Numbered Redactions: The same API key sk-ant-abc123... appearing three times becomes [REDACTED-API-KEY-1] everywhere. A different key becomes [REDACTED-API-KEY-2]. Readers can follow the structure of the conversation even in scrubbed transcripts.

Overlap Resolution: _resolve_overlaps() groups findings by position, sorts them, and when two overlap, keeps the higher-confidence (or longer) match. A real algorithm solving a real correctness problem.

Data Model: Enums for Confidence, ContentType, MessageRole. Dataclasses for ContentBlock, TranscriptMessage, TranscriptSession, Finding, RedactionResult. Every finding has precise char_start / char_end offsets.

What It Optimized For

  • 257 passing tests in 0.18s — round-trip verification
  • Stable numbered redactions preserving semantic coherence
  • Overlap resolution preventing double-match bugs
  • Built-in allowlists for safe IPs, domains, placeholder keys
  • Interactive review mode with allowlist persistence
  • Rich terminal UI with colored diffs, tables, panels

What It Sacrificed

  • Only 11 provider patterns vs alpha's 22
  • No HTML parser — cannot process HTML exports
  • No CI exit codes — scan always exits 0
  • Single file arguments only — no variadic args
  • TOML config only — no JSON config format
  • 4 runtime dependencies vs alpha's 1

The Asymmetry of Crossover

Alpha's additional patterns are essentially data (regex strings) that slot into beta's existing detector framework. But beta's core innovations (stable numbering, overlap resolution, the data model) would require fundamental restructuring of alpha's regex-sub-in-place architecture.

Alpha is a flat surface — easy to extend horizontally (add patterns) but difficult to extend vertically (add layers). Beta is a deep structure — easy to extend vertically (add detector classes) but each horizontal extension requires more ceremony.

The γ Roadmap (Ordered by Impact / Effort)

  1. Port α's 10 missing provider patterns into β's APIKeyDetector — each is a tuple of (name, regex, confidence, skip_allowlist). Immediate coverage gain.
  2. Add sys.exit(1) to β's scan command when findings > 0 — three lines. Unlocks CI/CD integration.
  3. Add nargs=-1 variadic files to scan and scrub commands — small CLI refactor, better ergonomics.
  4. Add string replacements to β's config and engine — post-redaction exact-match replacement for company/project names.
  5. Build HTMLParser(BaseParser) using α's tag-splitting approach — wrap in β's interface with can_parse(), parse(), reconstruct().
  6. Add disabled-by-default pattern support — a boolean enabled field in the detector config, mirroring α's Cloudflare handling.
